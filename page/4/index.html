alwayson database can not shrink log fileAlwaysOn database 事务日志截断后，日志文件无法收缩，导致日志文件占用磁盘空间大。 SolutionIf you have scheduled T-Log Backup on alwaysOn Database, After completion of T-Log Backup check the log_reuse_wait_desc of database using query 1select name,log_reuse_wait_desc from sys.databases where name like 'Name Of Database'; If column log_reuse_wait_desc returns value “NOTHING“, Immediately execute DBCC SHRINKFILE Command and shrink Log File without Error.
